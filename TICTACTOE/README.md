## 読む順番の目安

**初めて全体像を把握したい場合：**
1. `tictactoe_spec.md`（何を作るか）
2. `tictactoe_uml.md`（どう設計されているか）
3. `tictactoe_画面遷移仕様書.md`（実際の画面の流れ）
4. `tictactoe.c`（実装本体）
5. `tictactoe_テストケース仕様書.md`（実装が仕様通りかの確認観点）

**CPUの思考ルーチンだけを理解したい場合：**
1. `minimax法の解説.md`（考え方）
2. `getBestMove_マス目の例.md`（具体例）
3. `getBestMove_詳細解説.md` → `minimax_詳細解説.md`（実装の詳細）

**実装で読みづらい箇所だけをピンポイントで確認したい場合：**
- `tictactoe_わかりづらい部分の解説.md`

## 既知の制約

- `termios.h`（POSIX端末制御）を使用しているため、**Linux/macOSなどPOSIX準拠環境が前提**。Windowsの標準コンソールではそのままでは動作しない（詳細は`tictactoe_spec.md`の動作環境の章を参照）。
- 画面クリアにANSIエスケープシーケンスを使用しているため、対応していない端末では正しく表示されない場合がある。
